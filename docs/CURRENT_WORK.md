## Open threads

### LLM cooperation bias — ground agent objectives in real procurement KPIs

**Status:** in discussion, no implementation yet.

**The problem.** LLMs are RLHF'd to be cooperative, which leaks into agent behavior — agents concede too readily during negotiation, share information they shouldn't, and resolve conflict instead of sustaining it. This understates the realism of the crisis dynamics we're trying to emerge.

**Current mitigation (already in repo).** Hand-curated personas already include `INTERNAL DYNAMICS` (board/stakeholder tensions) and `YOUR KPIs` (4 numbered KPIs with concrete thresholds) — see `agents.py:_HARDCODED_PERSONAS`. The auto-generated FY2019 personas follow the same schema (`persona_builder.py`). KPIs include things like "gross margin >50%", "no single customer >30% of revenue", "premium-payment cap 25% above contract". This already does some of the work but isn't grounded in real procurement metrics.

**Proposed direction.** Use real procurement KPIs (PPV, OTD%, single-source exposure, savings vs. budget, DPO, supplier-risk score) as role-appropriate objective bundles per persona. Different agents, different objective functions, real cross-agent conflicts emerge. Separately, consider feeding ISM/PMI sub-indices (Supplier Deliveries, Prices Paid) as quarterly *prompt context* to ground sentiment in real data instead of narrative cues.

**Open questions for next session:**
- KPI source: auto-derive from the SEC filings that `persona_builder.py` already pulls (10-K KPI mentions, MD&A targets), or hand-curate per role?
- Success metric for "cooperation bias reduced": fewer concessions in negotiation, more walk-aways, higher price dispersion, something else? Pick before tuning.
- Scope: KPI grounding only, ISM/PMI prompt context only, or both?
